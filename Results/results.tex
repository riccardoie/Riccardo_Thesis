\chapter{Results}\label{chap:results}
This chapter presents the results of running METIS with NVOL. Throughout this chapter we will compare these results to METIS own total communication volume objective. To generate the graphs and plots in this chapter the tools described in Chapter~\ref{chap:NVOL tooling} were used. These results show the impact of applying a partition-aware objective function and compare whether this leads to any improvements in the resulting partitioning. 

\section{Graphs used for testing}
To evaluate NVOL we used symmetric matrices drawn from two datasets, treating each matrix as the adjacency representation of an undirected graph. Symmetry is required because the GPP, as defined in Section~\ref{sec:gpp}, is formulated on undirected graphs. The two datasets were chosen to cover a wide range of graph
sizes and to include instances arising from different application domains. We tried to prioritize graphs with a higher average degree as well. 

The first dataset is taken from \textit{Repository for modelling acute myocardial ischemia: simulation scripts and torso-heart mesh}~\cite{martinez-navarro2019a}, who simulate acute myocardial ischemia in a human heart using
the Chaste finite-element framework. The resulting meshes were then translated into a format accepted by METIS. Table~\ref{tab:heart_dataset} summarises the seven instances, ranging from roughly $10^3$ to almost $10^7$ vertices.

\section{Maximum outgoing volume per-partition}
Using the graphs listed above, we ran METIS with two objectives: NVOL and the standard total communication volume. For each run we then used
\texttt{calc\_partitions} to analyze the resulting partitions and compare their outgoing volumes. The number of partitions used for each graph depended on the
graph's size. To avoid splitting a graph into too many pieces, we required the square of the partition count to be at most equal to the number of vertices in the graph.

The following tables compare the maximum outgoing partition volume between METIS and NVOL. Using METIS as the baseline, we report each value as the percentage difference of NVOL relative to METIS. Cells are color-coded for readability: green indicates that NVOL achieves a lower maximum (better), while red indicates a higher maximum (worse). Shading intensity reflects the magnitude of the difference, with darker shades marking larger improvements or regressions.

Table~\ref{tab:heart_NVOLvsMETIS} shows the results from the Heart dataset. The first thing one notices is how NVOL is able to improve the maximum outgoing communication volume on all runs apart from 4 runs. Most of the time the improvement is between 3-5\% with a few peaks at 10\% and above. Like it was probably clear already from table~\ref{tab:heart_dataset}, these graphs are very similar in structure something that seems to translate in the partitions as well.
